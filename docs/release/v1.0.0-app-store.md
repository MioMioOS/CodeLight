## 4. App privacy questionnaire

This is the part Apple cares about most. **All answers below assume Code Light
collects nothing — which is true: there's no analytics, no tracking, no
telemetry, no third-party SDKs.**

### Data collection
**Q: Do you or your third-party partners collect data from this app?**
**A: No.**

That's it. The questionnaire short-circuits if you answer No here.

If Apple challenges this on review (they sometimes do for apps with network
features), justify with:

> Code Light syncs Claude Code session data between the user's own Mac and
> their own iPhone via a CodeLight Server that the user self-hosts (or that
> a community member hosts). Code Light, the app maker, does not operate
> any centralized server and does not have access to any user data. The
> device-to-device transport is end-to-end encrypted with the user's own
> Ed25519 keypair stored in the iOS Keychain. No analytics, no tracking,
> no third-party SDKs are integrated. Push notifications use APNs and the
> device token is sent to the user's chosen self-hosted server only.

## 9. Fresh-install test plan

Run this on a real iPhone before submitting. The reviewer's first launch is
the same path. **Do not skip.**

### Setup
1. Make sure your test Mac is running the latest MioIsland with the
   `feature/codelight-sync` branch built.
2. Get its current `shortCode` from MioIsland → notch menu → Pair iPhone.
3. On the iPhone, **delete the existing Code Light app entirely** (long-press
   icon → Remove App → Delete App). This resets all UserDefaults, Keychain
   tokens, and APNs registration.
4. Reinstall the latest build (`xcrun devicectl device install ...`).

### Walkthrough
- [ ] Launch app for first time. Should land on **Pair a Mac** screen.
- [ ] Tap **Enter Code** tab. Server URL field should be visible and prefilled.
- [ ] Tap quick-input pills (`https://`, `.chat` etc.) — should compose URL.
- [ ] Type the 6-char code from Mac. Tap **Pair**.
- [ ] Should see "**Paired with [Mac name]**" toast, then auto-navigate to
      **Macs** list. Should see 1 row.
- [ ] Tap into the Mac. Should see active sessions (or empty state).
- [ ] Tap a session → ChatView opens. Should see history streaming.
- [ ] Tap the **+** in toolbar → **LaunchSessionSheet**. Should see 2 default
      presets (`Claude (skip perms)`, `Claude + Chrome`) and recent projects.
- [ ] Pick a preset + project → tap **Launch**. Mac should open a new cmux
      window running the configured command.
- [ ] Back in chat, type a test message → tap send. Should appear in the
      Mac's terminal (typed into the Claude pane).
- [ ] Type `/model` → send. Should see the slash command response come back
      as a `terminal_output` message.
- [ ] Tap the **photo** button in compose bar. Menu should show **Take Photo**
      and **Choose from Library**.
- [ ] Tap **Take Photo**. iOS camera permission prompt should appear with
      the bilingual NSCameraUsageDescription (system language sensitive).
- [ ] Allow → take a photo → returns to chat with image attached.
- [ ] Tap send. Image should be pasted into the cmux pane on the Mac.
- [ ] Open **Settings** (gear icon top-left).
  - [ ] Servers section lists 1 server with green active indicator.
  - [ ] Paired Macs section lists 1 mac.
  - [ ] Tap **Reconnect** — should show running spinner → ✓ "已重新连接".
  - [ ] Tap **Cleanup Inactive Sessions** → confirm. Status should show 0 or N.
  - [ ] Tap **Privacy Policy** → should open in-app sheet (not GitHub).
- [ ] Force-quit the app, reopen. Should auto-connect and resume.
- [ ] Background the app, leave it for 1 minute, foreground again. Should
      reconnect cleanly and Live Activity should show current state.
- [ ] Lock the iPhone, send something on Mac. Live Activity on lock screen
      should update.
- [ ] **Long-press** Mac row in linked list, swipe → Unpair. Mac should be
      removed from list. Mac side: MioIsland should log `link-removed`.
- [ ] Re-pair using the same code. Should work (codes are permanent).

### Things that will get you rejected
1. **Crash on first launch** — usually missing `INFOPLIST_KEY_NSCameraUsageDescription` for any code path that touches the camera. We have it.
2. **Permission prompt without context** — we explain in-context (the menu
   item is "Take Photo", and the system description is bilingual).
3. **Privacy policy not viewable in-app** — we have PrivacyPolicyView.
4. **Login screen with no demo account** — we use device pairing, so no login
   screen exists. Reviewer needs the test Mac shortCode.
5. **Dead links** — make sure the GitHub URLs in About all resolve.

## 10. Post-submission

After tapping **Submit for Review** in App Store Connect:
- Apple's automated checks run for ~30 minutes
- Manual review queue takes 24-48 hours typically
- If rejected: read the rejection notes, fix, archive a new build with
  bumped `CURRENT_PROJECT_VERSION` (build number, NOT MARKETING_VERSION),
  upload, resubmit

For incremental releases after 1.0.0:
- Bump `CURRENT_PROJECT_VERSION` for every TestFlight upload
- Bump `MARKETING_VERSION` only for new public versions (1.0.1, 1.1.0, 2.0.0…)
